# frontend-design

## Purpose
Use this skill to make the MVP visually clearer and more professional for demos and real usage.

## Use when
- Improving dashboards or tables
- Refining forms and filters
- Making screens look more polished for stakeholders
- Improving visual hierarchy and usability

## Do not use when
- The task is primarily security hardening
- The task is schema or API design

## Project context
The MVP should be usable and presentable to business stakeholders, but visual work should not override security, data integrity, or delivery priorities.

## Workflow
1. Identify the primary user goal of the screen
2. Improve visual hierarchy
3. Clarify actions, states, and empty/loading feedback
4. Keep the UI professional and simple
5. Avoid purely decorative complexity

## Output requirements
Return:
- layout improvement suggestions
- hierarchy recommendations
- UX clarity notes
- optional component/UI ideas
